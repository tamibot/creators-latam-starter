No es qué instalás. Es cómo pensás el proyecto antes de tocar código.

Cualquiera puede clonar un starter y abrir Claude Code. El resultado: un proyecto que arranca rápido y se empantana. No hay proceso — solo herramientas.

Nosotros hacemos lo opuesto: primero pensamos, después ejecutamos. El Plan Maestro firmado es el contrato entre vos, el cliente y el sistema.

✓ ES

  • Una disciplina (qué, cómo, con qué)
  • Un contrato firmable con el cliente
  • Agentes que ejecutan un plan claro
  • Research obligatorio antes de elegir
  • Trabajo colaborativo por git

✗ NO ES

  • Un boilerplate de código
  • Un conjunto de plugins
  • Una receta "instalá y listo"
  • Una alternativa a pensar
  • Tecnología específica

⚠️ Bug que nos pasó

Si le pedís a un LLM imprimir un .md directo a PDF, queda feo: los Mermaid quedan como código, las tablas se cortan, sin tipografía. Solución: pasás por html-renderer primero, y el pdf-exporter valida visualmente antes de exportar.

💡 Tip

En VS Code / Cursor, si no se ve un diagrama Mermaid en un .md, presioná Cmd + Shift + V (Mac) o Ctrl + Shift + V (Win) para ver el preview renderizado. Así validás antes de exportar.

¿Y el dashboard interactivo?

Si lo que necesitás es un dashboard standalone (HTML con filtros, gráficos Chart.js, tablas ordenables — un archivo que el cliente abre en el navegador), usás el agente dashboard-builder directo. No pasa por PDF porque es interactivo.

Tip de onboarding

Cuando agregás un colaborador nuevo, invocá al agente mapa-sistema — te genera un documentation/mapa-sistema.md con todo lo que hay (agentes, skills, integraciones, custom). El colaborador lo lee en 10 min y está listo.

Instalación

3 comandos. Empezás a trabajar.

El instalador detecta qué ya tenés. Si todo está, sale en segundos. Si faltan cosas, las instala con tu autorización. 

# 1. Clonar el starter
$ git clone https://github.com/tamibot/creators-latam-starter.git mi-proyecto
$ cd mi-proyecto

# 2. Instalar herramientas (interactivo)
$ ./tooling/install.sh

# 3. Abrir Claude Code y arrancar con el plan
$ claude
> "Usá el skill plan-maestro para generar el scaffold, después invocá al agente kickoff-cliente"
TOOL 1

./tooling/doctor.sh

Diagnóstico del proyecto. Chequea todo y sugiere fixes. Correlo periódico.

TOOL 2

./tooling/validate-plan.sh

Verifica que las 3 fases del plan estén firmadas antes de ejecutar.

TOOL 3

./tooling/migrate.sh

Actualiza tu proyecto a la última versión del starter con git merge.

🛠️ Stack que viene preinstalado

El ./tooling/install.sh deja listo todo lo que usan los agentes. Si ya tenés algo, lo saltea. Si falta, lo instala con tu autorización.

Qué esperar

Timing realista. Sin sorpresas.

Cuánto tarda cada paso. Si algo dura más, algo está mal.

15–30 min

Primera instalación

Mac vacía: Homebrew + Python + Docker + marker (2GB) + whisper.cpp + CLIs. Largo la primera vez. No te asustes.

2–5 seg

Instalaciones posteriores

Detecta qué ya tenés y salta lo instalado.

30 min – 3 días

Paso 1 · Entregables

Según claridad del cliente. Idea vaga: 3 días. Con mockup: 30 min.

1–2 días

Paso 2 · Stack

Research en foros por cada herramienta. No se acelera.

4–8 horas

Paso 3 · Equipo + Paso 4 · Plan

Con los dos anteriores cerrados, armar equipo y Gantt es rápido.

FAQ

Preguntas frecuentes.

Forma de trabajo
¿Por qué "forma de trabajo" y no solo herramientas?
Porque 10 herramientas no arreglan la falta de proceso. El starter te da el proceso (4 pasos, 32 agentes, reglas) para que cualquier stack que uses después funcione.
¿Qué pasa si el cliente no quiere el Plan Maestro?
No trabajamos. Los proyectos sin plan firmado terminan mal para ambos. Mejor no arrancar.
¿Es obligatorio investigar en foros en el Paso 2?
Sí. Las docs oficiales mienten por omisión. Los foros te cuentan los problemas reales.
Setup
¿Cuánto tarda instalar?
Mac vacía: 15–30 min. Después: segundos (detecta qué ya tenés).
¿Necesito ANTHROPIC_API_KEY?
Si usás Claude Code: no. Usa tu sesión. Sólo si llamás la API desde código.
¿Corre en Windows?
El script asume macOS. Para Windows, pedile a Claude que adapte usando winget/scoop.
¿Puedo usar Cursor o Codex en lugar de Claude Code?
Sí. .claude/ es específico de Claude Code, pero CLAUDE.md y templates son universales.
Colaboración
¿Cómo trabaja un equipo con esto?
Invitás a GitHub, clonan, corren install.sh, abren Claude Code. Todos tienen los mismos 32 agentes. Trabajan en branches, abren PRs, el guardian-reglas audita antes del merge.
¿Cómo onboardear a un colaborador nuevo?
Invocás mapa-sistema, te genera documentation/mapa-sistema.md con todo lo que hay. El colaborador lo lee en 10 min.
¿Cómo manejamos permisos de quien puede hacer qué?
GitHub permissions + branch protection. Con protección de main, nadie mergea directo sin aprobación. El guardian-reglas valida las 10 reglas antes de mergear.
Troubleshooting
La instalación se colgó
Marker descarga ~2GB la primera vez. Si >30 min sin progreso: Ctrl+C, re-correlo. Detecta qué ya bajó.
El diagrama Mermaid en un MD no se ve renderizado
En VS Code / Cursor, presioná Cmd + Shift + V (Mac) o Ctrl + Shift + V (Win) para ver el preview renderizado del markdown.
Docker Desktop no arranca
Abrilo manualmente desde Applications una vez para aceptar permisos de macOS.
"command not found: brew"
echo 'eval "$(/opt/homebrew/bin/brew shellenv)"' >> ~/.zshrc
source ~/.zshrc
El agente no aparece en /agents
Reiniciá Claude Code. Chequeá que el frontmatter esté bien formado.
Doctor script muestra issues — cómo arreglo?
Correr ./tooling/doctor.sh --fix para correcciones seguras automáticas.
