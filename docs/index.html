FORMA DE TRABAJO · V1.5 · 2026

Esto no es una herramienta. Es una forma de trabajo.

Nuestro método de 3 fases para proyectos con IA. Primero definís qué entregás (desde el final). Después validás con qué lo hacés (investigando). Y recién ahí armás el equipo. Sin Plan Maestro firmado, no arranca nada.

Firmado antes de ejecutar

No es qué instalás. Es cómo pensás el proyecto antes de tocar código.

Cualquiera puede clonar un starter, instalar 10 herramientas y abrir Claude Code. El resultado de eso es un proyecto que arranca rápido y se empantana a la mitad. Porque cuando llega el problema, no hay un proceso que te saque — solo herramientas que no sabés qué hacen.

Nosotros hacemos lo opuesto: primero pensamos, después ejecutamos. El Plan Maestro firmado es el contrato entre vos, el cliente y el sistema. Con ese plan en mano, las herramientas son solo el cómo.

✓ ES

  • Una disciplina (qué, cómo, con qué)
  • Un contrato firmable con el cliente
  • Agentes que ejecutan un plan claro
  • Research obligatorio antes de elegir
  • Auditoría antes de cada entregable

✗ NO ES

  • Un boilerplate de código
  • Un conjunto de plugins
  • Una receta "instalá y listo"
  • Una alternativa a pensar
  • Tecnología específica (usa la que quieras)

La metodología

Tres fases. Ninguna se saltea.

Cada fase termina con validación explícita del cliente. No avanzamos hasta que la anterior esté firmada. Al final: Plan Maestro que el cliente aprueba.

1
FASE 1

Entregables · pensar desde el final

Trabajamos de adelante hacia atrás. Primero definimos qué archivo, pantalla o experiencia tiene que existir cuando terminemos. Después vemos cómo se produce.

¿Qué cosa concreta recibe el cliente cuando el proyecto termina perfecto?

Ejemplo real

Entregable: reporte-leads-abril.xlsx Ubicación: output/reportes/ Frecuencia: semanal, lunes 9am Columnas: · fecha # de Kommo API (lead.created_at) · nombre, email # de Kommo API (contact fields) · empresa # enriquecimiento con Clearbit · canal # de utm_source (HubSpot form) · estado # de Kommo pipeline stage · lifetime_value_usd # calculado: SUM(deals.amount) KPIs al final del reporte: · leads_totales # COUNT(*) · tasa_conversion # qualified / total · CAC # ad_spend / leads_convertidos · LTV:CAC # benchmark > 3:1 Aprobación: El cliente abre el Excel y dice: "sí, esto es lo que necesitaba."
✗ Anti-patrón

"Vamos a integrar la API de Kommo" → eso es stack, no entregable. Volvé al entregable primero.

✓ Validación
  • Formato y ubicación exactos por cada entregable
  • Cada columna/campo con fuente de datos trazable
  • KPIs con fórmula definida
  • Criterio de aprobación documentado
  • Cliente confirma por escrito antes de avanzar

Conectar herramientas

La lógica al sumar nuevas herramientas.

No usamos cualquier camino. Tenemos una jerarquía — siempre probamos la opción más robusta primero. El agente integrador-herramientas ejecuta esta lógica.

1
API REST oficial
Default siempre. Universal, documentada, versionada, auditable.
2
SDK oficial
Cuando existe y se mantiene activo, ahorra boilerplate.
3
MCP server
Si la herramienta tiene MCP, Claude Code la consume nativo. Ideal para flujos con LLM en el medio.
4
Webhook entrante/saliente
Para eventos en tiempo real. Requiere endpoint expuesto + validación HMAC.
5
n8n / Make / Zapier
Orquestación visual cuando hay muchos sistemas. Backup del workflow en workflows/.
6
UI automation (Playwright)
Último recurso. Se rompe con cada cambio de UI.

Regla

Para cada herramienta nueva, integrador-herramientas presenta al menos 2 alternativas con trade-offs explícitos. Nunca hay "la mejor" sin research — hay "la mejor para este caso, por estas razones".

Qué esperar

Timeline realista. Sin sorpresas.

Cuánto tarda cada paso en serio. Si algo dura más de lo indicado, algo está mal — no es normal.

15–30 min

Primera instalación del starter

En una Mac nueva sin nada: Homebrew + Python + ffmpeg + Docker + marker (modelos ~2GB) + whisper.cpp + CLIs modernos. Largo la primera vez, pero no te asustes — la siguiente corrida es en segundos.

2–5 seg

Instalaciones posteriores

El script audita qué tenés instalado y salta lo que ya existe. En un proyecto nuevo clonado, si ya usaste el starter antes, tarda segundos en validar.

30 min – 3 días

Fase 1 del Plan Maestro (Entregables)

Depende de qué tan claro tenga el cliente lo que quiere. Idea vaga: 3 días. Con screenshot del Excel que imagina: 30 min.

1–2 días

Fase 2 (Stack + research)

Research en foros por cada herramienta externa. No se acelera — acá evitás los sustos futuros.

4–8 horas

Fase 3 (Equipo + Gantt)

Con fases 1 y 2 cerradas, armar equipo y Gantt es rápido. El diagramador-mermaid te hace el Gantt en 15 min.

Variable

Ejecución (post-firma)

Recién acá arranca el código. El Gantt de la Fase 3 ya estableció las semanas. Ejecución pura.

¿Sentís que la instalación está colgada? Los modelos de marker tardan la primera vez (~2GB). Si llevás más de 30 minutos sin ver progreso, cancelá con Ctrl+C y correlo de nuevo — el script detecta qué ya bajó.
Comandos esenciales

Atajos de Claude Code que sí o sí tenés que usar.

Los que más nos salvan el día. Aprendelos de memoria — son la diferencia entre perder horas o avanzar tranquilo.

crítico
/compact

Compilar el contexto

Cuando la conversación llega cerca de 500k tokens, esto la resume y reinicia. Usalo antes de que el modelo empiece a inventar.

crítico
ESC ESC

Rewind (volver atrás)

Doble Escape te deja volver a un turno anterior. Cuando el modelo se equivocó y querés reintentar desde un punto limpio.

/clear

Limpiar el chat

Borra la conversación actual pero mantiene el proyecto abierto. Útil cuando el contexto se ensució.

/cost

Ver uso de tokens

Muestra cuántos tokens llevás usados en la sesión. Revisá antes de meter otro PDF grande.

/doctor

Health check

Diagnostica tu setup: Node, npm, MCP, permisos. Primer comando cuando algo "raro" pasa.

/agents

Gestionar agentes

Listá, creá, editá agentes desde Claude Code. Útil para crear los agentes custom de Fase 3.

/init

Generar CLAUDE.md

Si abrís un repo sin CLAUDE.md, te lo arma explorando el código.

/config

Settings

Abrir settings del IDE. Ajustar modelo, permisos, tema, idioma.

Atajos de teclado

Shift + Tab

Toggle plan mode

Entrás al modo planeamiento antes de ejecutar. El modelo te propone y vos aprobás.

Ctrl + R

Verbose

Ver más detalle de lo que hace el modelo. Útil cuando algo falla.

Ctrl + C

Cancelar

Para el comando actual. No cierra la sesión. Si el agente se va por la tangente, cortalo.

Mejores prácticas

Tips que aprendimos a la mala.

Reglas pequeñas pero firmes. La diferencia entre un proyecto que avanza y uno que se empantana.

Formatos y naming

📄

Markdown por default, para todo lo que Claude lea

Los modelos leen MD 10× más rápido que PDFs/DOCX/HTML. Todo PDF pasa por marker antes que el agente lo toque.

🏷️

Archivos con nombre descriptivo, sin espacios ni acentos

Usá reporte-leads-abril2026.xlsx, no "Reporte Leads (Abril 2026).xlsx". kebab-case siempre.

📂

Un cliente = una carpeta madre. Siempre.

Incluso tu propia agencia. "El contexto de la empresa" está todo junto. El modelo aterriza en segundos.

Cómo hablarle al modelo

🎯

Todo mensaje largo arranca con "Antes de comenzar, dime si tienes dudas"

El prompt del NV te ahorra 40 min de ejecución errada. Template en templates/nv-prompt.md.

🧭

Instrucciones claras > instrucciones extensas

5 bullets: objetivo, entrada, salida esperada, restricciones, criterio de corte. Si necesitás más, partí la tarea.

🛑

Si el modelo se va por la tangente, cortá con Ctrl+C

No dejes que se auto-corrija 5 veces quemando tokens. Cortá, reformulá, arrancá de nuevo.

Higiene del contexto

♻️

Compilá a los 500k tokens

/compact temprano es gratis; depurar errores del modelo confundido, no. Síntoma: el modelo se olvida de archivos que usaste hace rato.

🧹

Purgá la carpeta cada ciertos pasos

El agente archivero orquesta versionador + purgador. Carpeta limpia = contexto limpio.

🚫

Reemplazar > duplicar. Confiá en git.

Nada de plan_v2_final_FINAL.md. El versionador edita en sitio y git guarda el historial.

Seguridad

🔐

Credenciales: sólo en .env, nunca en código

El credentials-manager chequea antes de cada commit. Una credencial en un commit público = rotarla YA.

⛔

Si no usás Claude Code, no setées ANTHROPIC_API_KEY

Claude Code usa su propia sesión. Menos keys en el .env, menos superficie de ataque.

👀

Antes de cada commit grande: guardian-reglas

Audita las 10 reglas de oro. Si reporta crítico, no commites hasta resolver.

Metodología
¿Por qué arrancan con "Plan Maestro" y no directo con código?
Porque sin entregable definido, stack validado y equipo armado, el código que escribís es al pedo. El 80% de los proyectos fallidos que vimos empezaron con "dale, codeá". El Plan Maestro toma 2–5 días y ahorra semanas.
¿Qué pasa si el cliente no quiere pasar por las 3 fases?
Le explicamos que no trabajamos sin plan firmado. Si insiste, agradecemos pero no tomamos el proyecto.
¿Es obligatorio investigar en foros en la Fase 2?
Sí. Las docs oficiales siempre están incompletas o desactualizadas. Los foros te muestran los problemas reales. Documentado en documentation/stack/.
Setup e instalación
¿Cuánto tarda instalar todo?
Primera vez en Mac vacía: 15–30 min. En corridas siguientes: segundos — el instalador detecta qué ya tenés y salta lo instalado.
Necesito ANTHROPIC_API_KEY?
Si usás Claude Code, no. Usa tu sesión de login. Solo la necesitás si llamás la API desde código.
¿Corre en Windows?
El script asume macOS. Para Windows, pedile a Claude/Cursor que aplique lo mismo con winget o scoop.
¿Qué hace .claude/settings.json?
Define qué comandos puede ejecutar Claude Code sin pedirte autorización cada vez. Lista blanca para seguros, deny crítico para destructivos (rm -rf, force-push, leer .env).
¿Puedo usar Cursor o Codex en vez de Claude Code?
Sí. La estructura de .claude/ es específica de Claude Code, pero reglas y templates son universales.
Agentes y ejecución
¿Puedo armar agentes custom para mi proyecto?
Sí — el onboarding-pm los propone en Fase 3. Creás un .md en .claude/agents/ con el frontmatter estándar.
¿Cuánto cuesta en tokens una sesión típica?
Fase 1 (entregables): ~20k-50k input. Fase 3 con generación de código: 100k-500k. Usá /cost para ver en tiempo real.
¿Qué hago si un agente se va por la tangente?
1) Ctrl+C para cortar. 2) ESC ESC para volver al turno anterior. 3) Reformulá siguiendo el NV (objetivo, entrada, salida, restricciones, corte).
¿Cómo actualizo el starter cuando sacamos versión nueva?
git remote add starter https://github.com/tamibot/creators-latam-starter.git
git fetch starter
git merge starter/main --allow-unrelated-histories
El versionador ayuda con conflictos.
¿Qué diferencia tiene con starters de Vercel/Next?
Aquellos son templates de código. Éste es un starter de proceso: reglas, agentes, metodología. Se monta sobre cualquier stack.
Troubleshooting
La instalación se colgó — ¿qué hago?
Los modelos de marker tardan la primera vez (~2GB). Si >30 min sin progreso: Ctrl+C y re-correlo. Detecta qué ya bajó.
El modelo no encuentra un archivo que yo veo
Problema de contexto. /clear o /compact y re-mencionar con ruta completa. Si persiste, chequeá permisos en .claude/settings.json.
Docker Desktop no arranca después de instalar
Abrilo manualmente desde Applications al menos una vez — pide permisos macOS que no pueden darse automáticamente.
"command not found: brew" después de instalar
Agregá Homebrew al PATH:
echo 'eval "$(/opt/homebrew/bin/brew shellenv)"' >> ~/.zshrc
source ~/.zshrc
"externally-managed-environment" al instalar con pip
Python 3.12 protege el sistema. Usá uv pip install --system (incluido) o pip3 install --user --break-system-packages.
El agente no aparece en /agents
Reiniciá Claude Code. Si aún no aparece, chequeá el frontmatter (---, name:, description:, ---).
