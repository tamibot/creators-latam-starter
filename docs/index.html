PASO 00 · INSTALAR

Instalá la base.

Una sola vez. Con este paso quedás con Claude Code abierto en una carpeta del escritorio y los 32 agentes listos para usar.

1 Abrí Claude Code en tu Mac. Si no lo tenés, bajalo de claude.com/download.
2 Creá una carpeta en tu Escritorio exclusiva para este proyecto. Ej: ~/Desktop/mi-proyecto.
3 En Claude Code, abrí esa carpeta (cd o drag-and-drop), copiá el comando de la derecha y pegalo en el chat de Claude.
4 Dale todos los permisos que te pida Claude. Son seguros — el instalador es auditable y open source.

PREGUNTAS FRECUENTES

Lo que más preguntan.

¿Necesito ser técnico para usar esto?
No. El flujo está pensado para que cualquiera pueda ejecutarlo en Claude Code. El Paso 00 es una sola instrucción que copias y pegas. Los pasos 01-04 son guiados — el sistema te hace preguntas, vos respondés (incluso por nota de voz).
¿Qué es Claude Code y cómo lo instalo?
Claude Code es el IDE oficial de Anthropic para trabajar con Claude. Se descarga de claude.com/download. Requiere cuenta en Anthropic (gratis para empezar).
¿Cuánto tarda la instalación?
Primera vez en una Mac vacía: 15–30 min. Descarga herramientas (Python, Node, Docker), modelos de IA (~2GB) y configura todo. Siguientes corridas: segundos — detecta qué tenés y salta lo instalado.
¿Funciona en Windows y Linux?
El installer asume macOS. En Windows pedile a Claude que adapte usando winget o scoop. En Linux, apt o dnf. Las herramientas subyacentes existen en todas las plataformas.
¿Necesito Anthropic API Key?
Si usás Claude Code como IDE, no — usa tu sesión de login. Solo la necesitás si vas a hacer llamadas a la API desde código externo.
¿Puedo agregar mis propios agentes?
Sí — es el Paso 03. Corré ./tooling/new-agent.sh (wizard interactivo). Valida nombre único, frontmatter correcto, y queda integrado con los 32 de base.
¿Puedo trabajar en equipo con otros devs?
Sí. Invitás colaboradores al repo de GitHub. Cada uno clona, corre install y tiene los mismos 32 agentes. El guardian de reglas audita antes de cada merge.
¿Qué pasa si se rompe algo en medio del proyecto?
Hay un agente rollback-manager que te lleva al último commit verde (validado por CI). El script ./tooling/doctor.sh --fix diagnostica y repara automáticamente lo que puede.
¿Es open source? ¿Gratis?
Sí, 100% open source con licencia MIT. Usalo, clonalo, modificalo. Si encontrás mejoras generales, proponé un PR y las subimos al starter base.
